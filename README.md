# Atividade 1: Relatório e Diretrizes

## Conteúdo do Relatório
1. **Descrição da Solução:** Explique como seu servidor gerencia a lista de clientes ativos e como o `Lock` protege o estoque.
2. **Resultados do Stress Test:** Prints mostrando que o estoque nunca ficou abaixo de zero, mesmo com 30+ threads tentando comprar simultaneamente.
3. **Análise do Broadcast:** Evidência de que clientes que estavam apenas "conectados" receberam o aviso de esgotamento.

## Uso de Inteligência Artificial
O uso de IA é permitido para auxílio técnico, desde que reportado de forma ética:
- **O que declarar:** Qual ferramenta foi usada e para qual finalidade (ex: "ajuda para debugar o erro de timeout no socket").
- **Crítica:** Identifique uma sugestão da IA que você precisou modificar para que o código funcionasse no laboratório Linux.
- **Responsabilidade:** O grupo deve ser capaz de explicar o funcionamento total do código durante a arguição. O "copy-paste" sem entendimento resultará em nota zero.
